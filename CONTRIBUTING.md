# Contributing

Thank you for your interest in contributing to AI Agent Maintainer Kit.

This project is early-stage. Contributions are welcome, especially improvements that make AI-assisted maintenance safer, clearer, and easier for small open-source projects.

## Contribution areas

Useful contributions include:

- clearer template wording,
- safer default rules,
- example workflows,
- documentation improvements,
- maintainer checklists,
- tool-specific usage notes,
- issue triage improvements.

## Project principles

Please keep contributions aligned with these principles:

1. Human maintainers stay in control.
2. AI workers must have explicit task boundaries.
3. Sensitive data must not be exposed in prompts, reports, logs, or examples.
4. Git write operations require explicit human permission.
5. Templates should be easy to copy into small repositories.
6. Examples should be generic and should not include real private project data.

## How to contribute

1. Open an issue describing the change or improvement.
2. Keep pull requests small and focused.
3. Explain what problem the change solves.
4. Avoid adding project-specific private assumptions.
5. Update examples or documentation when needed.

## Safety review

Changes that affect approval rules, Git operation rules, or sensitive data handling should be reviewed carefully.

When in doubt, choose the safer default.
